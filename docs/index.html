What is a Custom Widget?

The most powerful widget type in Streamlabs — a blank canvas with full access to stream events.

No restrictions. Unlike other Streamlabs widgets, the Custom Widget has no built-in rendering layer. You write the HTML, CSS, and JS from scratch. It receives stream events and you decide exactly what to do with them.
📡

Listens to any stream event. Follows, subs, donations, bits, raids, merch, and more — all delivered via onEventReceived.

🎨

Fully custom visuals. No Vue, no widget framework. Your HTML and CSS render exactly as written.

📦

Load any library. The Custom Widget sandbox allows external CDN scripts — use GSAP, Three.js, Chart.js, whatever you need.

💾

Persistent state. Unlike alert boxes, the widget stays alive between events. You manage your own state.

⚠️

No built-in queue. Events arrive in real time as they happen — simultaneously and not necessarily in order. If you need to display events one at a time, you must build your own queue. See the examples for a working implementation.

How to use

No build step, no install.

Download

Grab the ZIP and extract it anywhere on your machine.

Pick an example

Open the folder for the example you want. Read its readme.md for options and tokens.

Add a Custom Widget

In Streamlabs, add a Custom Widget source to your scene and open its settings.

Paste

Copy the contents of each file into the matching Custom HTML, Custom CSS, and Custom JS fields. Save and you're live.

Good to know

🌐

Chrome only. Streamlabs Desktop and OBS embed a Chromium-based browser source.

📦

jQuery is not pre-loaded in Custom Widgets — unlike AlertBox and other widgets. Each example either includes it or uses vanilla JS.

🔤

Field data tokens like {field_name} are replaced at runtime from your widget's Fields tab settings. They won't resolve in local preview.

🧪

Testing locally. Use document.dispatchEvent calls in your browser console to simulate stream events. See the Starter Kit for fake event snippets.
